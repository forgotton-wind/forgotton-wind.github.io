Github连接问题

Github连接注意事项 在git push或者pull的时候，连接github总是遇到各种各样的网络问题，现在一次性总结一下，方便以后回顾。 网络问题 很多时候直接连不上，会先简单挂个梯子。但是往往未必有用。可以尝试两个方法。 开全局模式。可能是github不一定走代理，毕竟有时候还是能直连的。 单独配置git的代理端口。配置方法是 git config --global http.proxy "http://127.0.0.1:xxxx" git config --global https.proxy "https://127.0.0.1:xxxx" 其中xxxx是你的代理的端口号，windows可以通过网络->代理那边去查看。 配置完后可以检查下： git config --global --edit 看看对不对，有时候字符会有些问题。 密码问题 git push的时候，以前是可以输入github的账号密码的，但是现在都需要输入个人秘钥。这个需要再github的个人主页的settings里面去创建。 但是这个方法很麻烦，每次git push都要输入一次秘钥。一劳永逸的方法是配置ssh连接。但是这个里面也有一些坑。 首先配置ssh连接需要创建本机的ssh key： ssh-keygen -t rsa -C "你的git绑定的邮箱名字"
